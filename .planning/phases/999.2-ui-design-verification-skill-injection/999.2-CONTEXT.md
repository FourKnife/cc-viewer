**Gathered:** 2026-04-28
**Status:** Ready for planning

<domain>
## Phase Boundary

在"结构化对比设计稿"按钮触发的还原流程中，自动检测并注入 `ui-design-verification` skill 内容到发送给 Claude Code 的 prompt 里。若 skill 未安装，内联展示安装命令引导用户。

</domain>

<decisions>
## Implementation Decisions

### 注入方式
- **D-01:** 采用方案 A — 将 skill 内容直接嵌入 prompt，不写入目标项目的 CLAUDE.md。
- **D-02:** 原因：写 CLAUDE.md 会改变系统 prompt，破坏 KV cache。

### 触发点
- **D-03:** 复用现有 `handleAutoCompare`（`PagePreview.jsx:452`），不新增按钮。
- **D-04:** 按钮文案已有 i18n key `visual.structCompare.*`，UI 无需变动。

### Skill 内容加载
- **D-05:** 运行时读取 SKILL.md，不硬编码快照到 server.js。skill 更新即生效。
- **D-06:** 使用现有 `listSkills()` from `lib/skills-api.js` 检测 skill 是否安装，获取 skill path，再读取 `SKILL.md` 内容。
- **D-07:** 新增 server 端点 `GET /api/skill-content?name=ui-design-verification`，返回 `{ found: true, content: "..." }` 或 `{ found: false }`。

### Skill 未安装时的处理
- **D-08:** 内联展示安装命令，不弹窗。在操作区直接显示：
  ```
  ui-design-verification skill 未安装
  tnpx @antskill/agentic-installer @antskill/ui-design-verification
  ```
- **D-09:** 安装命令需可一键复制。

### Prompt 拼接
- **D-10:** skill 内容前置拼接在 `handleAutoCompare` 构建的结构化 diff prompt 之前（`PagePreview.jsx:514` 前插入）。
- **D-11:** skill 内容与原 prompt 之间用空行分隔，保持可读性。

</decisions>
