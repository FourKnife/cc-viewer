# M1.5: 项目启动器 UI 改造 — 需求文档

**版本**: v1.5
**创建时间**: 2026-04-24

---

## 背景

M1.4 完成后，可视化编辑器核心功能趋于稳定。用户在日常使用中暴露出项目启动器的三个体验问题：

1. **日志噪音**：启动日志包含大量 ANSI 转义序列（`[32m`、`[2K` 等），实际内容被噪音淹没，用户需要长时间滚动
2. **Available Pages 访问低效**：日志中的 Available Pages 列表需要手动复制 URL 粘贴到 URL 输入框，流程繁琐
3. **PagePreview 依赖启动器**：UI 编辑模式下，URL 输入框仅在项目启动后才显示，用户从外部启动的项目无法直接填入 URL

---

## 需求

### R1: ProjectLauncher 迁移至 SideMenu

- **现状**：ProjectLauncher 在底部 BottomTabPanel 的「项目启动器」Tab 中，空间受限（约 220px 高），大量日志无法舒适浏览
- **期望**：将「项目启动器」作为 SideMenu 的独立菜单项（与「UI 编辑」并列）。点击后整个 center 区域渲染 ProjectLauncher，拥有充裕的日志展示空间
- **验收**：
  - SideMenu 新增「项目启动器」菜单项（推荐 icon：`RocketOutlined` 或 `ThunderboltOutlined`）
  - 点击后 center 区域切换为 ProjectLauncher，日志区域占满可用高度
  - 项目启动/停止功能正常
  - BottomTabPanel 的「项目启动器」Tab 移除

### R2: PagePreview 常态化显示

- **现状**：`PagePreview.jsx` 在 `!port && !iframeSrc` 条件下显示空状态（仅图标 + 提示文字），不渲染 URL 输入框
- **期望**：UI 编辑模式下 URL 输入框 + iframe 区域始终可见，用户可直接输入任意 URL 无需先启动内置项目
- **验收**：
  - 无论 `port` 是否存在，URL 输入框和 iframe 区域始终渲染
  - 用户输入 `http://localhost:3002/demo.html` 后按回车，iframe 正常加载
  - 项目启动后自动填入端口 URL 的逻辑保留（不破坏现有行为）

### R3: ANSI Escape Codes 过滤

- **现状**：日志内容含有大量 ANSI 转义序列，如 `[32m`、`[2K [1A [2K [G`、`[36m⠸[39m` 等，严重降低可读性
- **期望**：在 ProjectLauncher 渲染日志前自动过滤所有 ANSI 转义序列，只展示纯文本内容
- **验收**：
  - 日志中不再出现 `[32m`、`[2K`、`[1A`、`[G` 等字符串
  - Spinner 进度行（`⠸ Compiling...`）被过滤后不残留空行
  - 正常文字内容（如 `Compiled successfully`、`Available Pages:` 等）保留完整

### R4: Available Pages 快捷导航

- **现状**：启动日志包含 Available Pages 列表，用户需手动复制 URL 到地址栏
- **期望**：解析日志中的 Available Pages 块，以可点击按钮形式展示。点击后自动填入 `previewUrl` 并切换到 UI 编辑菜单
- **格式参考**：
  ```
  Available Pages:
  - demo:    http://localhost:3002/demo.html
  - transferInCKK:  http://localhost:3002/transferInCKK.html
  ```
- **验收**：
  - 项目启动后，ProjectLauncher 区域出现解析到的页面列表
  - 每个页面显示名称 + URL，点击后 `previewUrl` 更新 + 切换到「UI 编辑」模式
  - 页面列表持久化（切换到 ui-edit 再切回 launcher，列表依然存在）

## 约束

- Anti-AI-Slop 原则：不引入多余的视觉装饰，布局简洁
- 不破坏现有功能：元素选择、Sketch 对比、TerminalPanel 等保持不变
- BottomTabPanel 的「元素信息」Tab 保留（仍承担选中元素信息展示职责）
- 测试全部通过
