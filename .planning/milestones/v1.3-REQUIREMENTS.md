# Cleffa v1.3 — 需求文档

**里程碑**: M1.3 - 优化可视区域
**状态**: 进行中
**目标**: 重构可视化编辑器布局为「左侧菜单 + 中间预览/操作 + 右侧终端」三栏结构，提升专业感和可扩展性

## 功能需求

### P0 - 必须完成

- [ ] **FR-301: 左侧功能菜单**
  - 将当前左侧 sidebar（ProjectLauncher + ElementInfo 纵向堆叠）替换为图标式功能菜单
  - 菜单项 1：「UI编辑」— 当前已实现的选中 DOM 与 CC 对话功能（默认激活）
  - 菜单项 2：「Pipeline」— 选择 Bakery mock 数据测试场景组，支持录入操作，模拟截图留痕（**仅占位，不实现功能**）
  - 菜单宽度紧凑（图标+文字或纯图标），不占用过多横向空间

- [ ] **FR-302: 中间区域上下分割**
  - 上半部分：页面预览区（iframe + url bar），占据主要空间
  - 下半部分：操作区，包含 ProjectLauncher 和 ElementInfo
  - 状态栏（StatusBar）移到中间区域顶部（预览区上方）
  - 上下区域支持拖拽调整比例

- [ ] **FR-303: 操作区折叠/展开**
  - ProjectLauncher 支持折叠/最小化到底部（仅显示一行状态摘要）
  - 项目启动后，ProjectLauncher 自动折叠，最大化预览空间
  - 点击可重新展开

### P1 - 应该完成

- [ ] **FR-304: Pipeline 占位菜单**
  - 左侧菜单中 Pipeline 项可点击，切换中间区域显示占位内容
  - 占位内容：简单说明文字，表明功能即将推出
  - 不实现实际的 Bakery mock 场景选择和操作录入功能

## 非功能需求

- [ ] **NFR-301: 遵循 Anti-AI-Slop 设计原则** — 不使用紫色渐变、emoji 图标、圆角卡片+左侧色条等 AI 默认风格。使用项目已有 CSS 变量，不新增颜色
- [ ] **NFR-302: 向后兼容** — 不破坏 chat/raw 视图模式和 M1.0-M1.2 已有功能
- [ ] **NFR-303: 右侧终端不变** — 右侧 xterm 终端面板保持 400px 宽度和现有功能
- [ ] **NFR-304: 布局响应** — 中间区域宽度自适应（flex: 1），左侧菜单和右侧终端固定宽度
