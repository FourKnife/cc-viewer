## Phase 21: 日志优化 + Available Pages 快捷导航

**目标**: 过滤 ANSI escape codes；解析 Available Pages 并展示为可点击快捷入口

**任务**:
- [ ] 实现 `stripAnsi(text)` 工具函数（`src/utils/stripAnsi.js`），覆盖：
  - CSI 序列：`\x1b[...m`、`\x1b[nA/B/K/G` 等光标/清屏控制
  - OSC 序列：`\x1b]...\x07`
  - 其他 ESC 序列
- [ ] `ProjectLauncher.jsx` — 渲染日志前调用 `stripAnsi()`，同时过滤由 spinner/进度行产生的空行
- [ ] 实现 `parseAvailablePages(text)` 工具函数，解析日志中 `Available Pages:` 块，返回 `[{ name, url }]`
- [ ] `App.jsx` — 新增 `availablePages` state，监听 `projectOutput` 变化时更新（`parseAvailablePages`）
- [ ] `ProjectLauncher.jsx` — 接收 `availablePages` prop，项目启动后在日志上方/下方展示页面快捷按钮列表
- [ ] 点击页面按钮：调用 `onPreviewUrlChange(url)` + `onSelectMenu('ui-edit')`（需从 App 传入 callback）
- [ ] 新增 i18n 键 `visual.launcher.pages`（"可用页面"）

**关键文件**:
- `src/utils/stripAnsi.js`（新建）
- `src/utils/parseAvailablePages.js`（新建）
- `src/components/VisualEditor/ProjectLauncher.jsx`
- `src/App.jsx`
- `src/i18n.js`

**预计改动量**: 中

**依赖**: Phase 20（ProjectLauncher 已在独立视图中，日志区域充裕）

## 里程碑成功标准

1. SideMenu 出现「项目启动器」菜单项，点击后 center 切换到 ProjectLauncher 全高视图
2. UI 编辑模式下，未启动项目时 URL 输入框仍可见，可直接输入外部 URL
3. 日志中不再出现 `[32m`、`[2K` 等 ANSI 字符
4. 项目启动后，Available Pages 以按钮形式出现，点击自动导航到 UI 编辑并填入 URL
5. 全部测试通过
