## Phase 14: 可视化编辑器布局重构

**目标**: 将现有三栏布局（左侧sidebar | 中间预览 | 右侧终端）重构为（左侧菜单 | 中间上下分割 | 右侧终端）

**任务**:
- [ ] 新建左侧功能菜单组件 `SideMenu.jsx`，包含「UI编辑」和「Pipeline」两个菜单项
- [ ] 「UI编辑」默认激活，「Pipeline」仅占位（点击显示 coming soon）
- [ ] 重构 `App.jsx` 中 `viewMode === 'visual'` 的布局结构：
  - 左侧：SideMenu（窄栏，约 48-60px）
  - 中间：上方 StatusBar + 页面预览区（上）+ 操作区（下，含 ProjectLauncher + ElementInfo）
  - 右侧：TerminalPanel（400px，保持不变）
- [ ] StatusBar 从底部移到中间区域顶部
- [ ] 中间区域上下分割支持拖拽调整比例（复用 PanelResizer 思路，改为水平方向）
- [ ] 更新 `App.module.css` 和 `styles.module.css` 中的 visual 布局样式
- [ ] 添加对应 i18n 条目

**关键文件**:
- 新建: `src/components/VisualEditor/SideMenu.jsx`, `src/components/VisualEditor/SideMenu.module.css`
- 修改: `src/App.jsx`（visual 模式布局区域）, `src/App.module.css`
- 修改: `src/components/VisualEditor/styles.module.css`
- 修改: `src/i18n.js`

**预计改动量**: 大

**设计约束**:
- 遵循 Anti-AI-Slop 原则：不使用紫色渐变、emoji 图标、左侧色条装饰
- 使用项目已有 CSS 变量（`var(--bg-*)`, `var(--text-*)`, `var(--border-*)`）
- 菜单图标使用 antd Icons 或纯 SVG，风格简洁
- 保持与现有深色/浅色主题兼容

## Phase 15: 操作区折叠 + UI 精细化

**目标**: ProjectLauncher 支持折叠/最小化，整体 UI 按设计规范打磨

**任务**:

- [ ] 审查并移除 Anti-AI-Slop 违规元素（如有）
- [ ] 检查深色/浅色主题下的视觉一致性

**关键文件**:

- 修改: `src/components/VisualEditor/styles.module.css`

**预计改动量**: 中

**依赖**: Phase 14（布局结构完成后才能做折叠优化）

## 里程碑成功标准

1. 可视化编辑模式左侧是紧凑的功能菜单，「UI编辑」默认激活，「Pipeline」显示占位
2. 中间区域上方为状态栏+页面预览，下方为操作区（ProjectLauncher + ElementInfo），上下可拖拽调整
3. 项目启动后 ProjectLauncher 自动折叠，预览区域最大化
4. 右侧 xterm 终端保持完整功能不变
5. UI 不含 AI-Slop 痕迹（无紫色渐变、无 emoji 图标、无圆角卡片+色条装饰）
