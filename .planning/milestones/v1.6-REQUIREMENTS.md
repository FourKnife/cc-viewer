# M1.6: Scenario 场景系统 — 需求文档

**版本**: v1.6
**创建时间**: 2026-04-27

---

## 背景

M1.5 完成后，可视化编辑器已具备元素选择、Sketch 对比、URL 导航等核心能力。但开发者在调试多页面、多状态的应用时，仍需手动导航到每个页面状态（如：登录态、空数据态、错误态、特定 Tab 激活态），效率低下。

核心痛点：
1. **状态难以复现**：每次刷新后需要重新手动操作才能到达目标状态
2. **多状态对比困难**：无法一次性看到所有页面状态的截图
3. **调试流程割裂**：在 Sketch 对比时，需要先手动导航到对应状态，再截图对比

---

## 需求

### R1: Scenario 数据层

- **定义**：Scenario 是一个命名的页面状态描述，包含：目标 URL、需要注入的 localStorage/sessionStorage 数据、可选的交互步骤序列
- **存储**：保存在项目目录下的 `.cleffa/scenarios.json`，可纳入 git 管理，团队共享
- **数据结构**：
  ```json
  {
    "scenarios": [
      {
        "id": "uuid",
        "name": "购物车 - 有商品",
        "url": "/cart",
        "storage": {
          "auth_token": "mock_token_123",
          "cart_items": "[{\"id\":1,\"qty\":2}]"
        },
        "steps": [
          { "type": "click", "selector": ".tab-detail" },
          { "type": "wait", "ms": 500 }
        ]
      }
    ]
  }
  ```
- **验收**：
  - `.cleffa/scenarios.json` 在项目目录下自动创建（首次保存时）
  - 增删改 scenario 后文件实时更新
  - 重启 cc-viewer 后数据持久化

### R2: Scenario 面板 UI

- **入口**：SideMenu 新增「场景」菜单项（icon: `ExperimentOutlined`，i18n key: `visual.menuScenarios`）
- **面板功能**：
  - 列表展示所有 scenario（名称 + URL）
  - 新增 scenario（弹窗表单：名称、URL、storage 键值对编辑器）
  - 编辑 / 删除 scenario
  - 运行按钮：执行 scenario（注入 storage + 导航 + 执行 steps）
  - 固定按钮：将 scenario 固定到 UI 编辑模式（刷新自动重跑）
- **验收**：
  - 面板可正常增删改 scenario
  - 运行按钮触发后，iframe 自动导航到目标 URL 并注入 storage
  - 固定状态在 SideMenu 图标上有视觉标记

### R3: 基础运行（Storage 注入 + 导航）

- **机制**：利用代理架构的同源特性，在导航前直接写入 `iframe.contentWindow.localStorage`
- **执行顺序**：
  1. 清除旧 storage（可选，避免污染）
  2. 注入 scenario.storage 中的所有键值
  3. 调用 `handleNavigate(scenario.url)` 导航
- **验收**：
  - 运行 scenario 后，iframe 加载目标页面，页面能读取到注入的 localStorage 数据
  - 支持相对路径（`/cart`）和完整 URL（`http://localhost:3001/cart`）

### R4: Steps 执行引擎（click/wait/fill）

- **机制**：扩展现有 inspector postMessage 协议，父页面发送 `run-step` 指令，inspector 脚本执行后回传 `step-done`
- **支持的步骤类型**：
  - `click`: `{ type: "click", selector: ".tab-detail" }` — querySelector + dispatchEvent
  - `wait`: `{ type: "wait", ms: 500 }` — setTimeout
  - `fill`: `{ type: "fill", selector: "input.search", value: "hello" }` — nativeInputValueSetter + input event（兼容 React 受控组件）
- **执行状态**：面板显示当前执行进度（步骤 N/M）
- **验收**：
  - click 步骤能触发 React 合成事件（Tab 切换等）
  - wait 步骤正确等待指定毫秒数
  - fill 步骤能更新 React 受控 input 的值
  - 步骤执行失败时（selector 未找到）给出警告，不中断后续步骤

### R5: 批量截图 + 状态画廊

- **功能**：一键运行所有 scenario，每个 scenario 执行完成后自动截图，最终展示画廊视图
- **画廊视图**：网格布局，每格显示 scenario 名称 + 截图缩略图，点击可放大
- **验收**：
  - 批量运行按钮触发后，逐个执行所有 scenario 并截图
  - 画廊视图正确展示所有截图
  - 截图可下载（单张 / 全部打包）

### R6: 固定 Scenario（刷新自动重跑）

- **功能**：在 UI 编辑模式下，可将某个 scenario 设为「固定」。每次 iframe 刷新（手动刷新或代码热更新触发）后，自动重新执行该 scenario
- **状态标记**：固定的 scenario 在面板列表中有高亮标记；URL 栏附近显示「已固定场景: xxx」提示
- **验收**：
  - 固定 scenario 后，手动点击刷新按钮，iframe 重新加载并自动执行 scenario
  - 取消固定后，刷新不再自动执行
  - 固定状态在 App state 中持久化（切换菜单再切回不丢失）

---

## 约束

- Anti-AI-Slop 原则：UI 简洁，不引入多余装饰
- 不破坏现有功能：元素选择、Sketch 对比、URL 导航等保持不变
- inspector 脚本扩展需向后兼容（旧消息类型继续工作）
- 测试全部通过
