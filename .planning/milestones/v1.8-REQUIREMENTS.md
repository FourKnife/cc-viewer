# M1.8: 可视化编辑-场景优化 — 需求文档

**版本**: v1.8
**创建时间**: 2026-04-28

---

## 背景

M1.6 和 M1.7 完成了 Scenario 场景系统的基础建设（CRUD、执行引擎、录制、AI 生成、批量截图）。但在日常使用中，以下痛点逐渐显现：

1. **Step 类型受限**：仅支持 click/wait/fill，无法覆盖 scroll、键盘操作、元素断言等常见调试场景
2. **Selector 编辑低效**：需要手动输入 CSS selector，缺少可视化拾取能力
3. **录制能力单一**：录制不能暂停/恢复，无法在录制中插入步骤
4. **场景与预览割裂**：场景面板独占全屏，无法边预览边编辑场景
5. **AI 生成不可控**：模型固定、无法精炼已有 step、不支持新增的 step 类型

本里程碑聚焦「Step 编辑器增强 + 录制体验优化」两大方向，同时将场景面板整合到 Visual Editor 底部 Tab 中。

---

## 需求

### R1: 底部 Tab 集成 + 场景面板重构

- **目标**：将 ScenarioPanel 从全屏独占视图迁移到底部 Tab 面板，支持边预览边编辑场景
- **方案**：
  - BottomTabPanel 新增「场景」Tab（与现有的「Element Info」Tab 并列）
  - 场景 Tab 内容：精简版 ScenarioPanel（核心功能：运行、固定、step 编辑）
  - 保留全屏场景视图入口（用于批量运行、画廊等复杂操作）
  - Tab 切换时不丢失场景编辑状态
- **验收**：
  - ui-edit 模式下底部 Tab 可切换到场景面板
  - 场景 Tab 中可编辑 steps、运行、固定
  - 切换到 Element Info Tab 再切回，场景编辑状态不丢失
  - 全屏场景视图入口正常（SideMenu「场景」项）
  - 现有 Scenario 功能完全保留

### R2: Step 编辑器全面增强

- **目标**：扩展 step 类型体系，提供可视化的元素选择器和 step 管理能力
- **新增 step 类型**：
  - `scroll`: `{ type: "scroll", selector: ".target" }` — 滚动到指定元素可见
  - `keyboard`: `{ type: "keyboard", value: "Enter" }` — 发送键盘按键（Enter/Tab/Escape/箭头等）
  - `hover`: `{ type: "hover", selector: ".menu-item" }` — 鼠标悬停触发 hover 状态
  - `select`: `{ type: "select", selector: "select.dd", value: "option-value" }` — 选择 `<select>` 选项
  - `assert`: `{ type: "assert", assertion: "element-exists"|"element-not-exists"|"text-matches", selector, expected }` — 断言元素存在/不存在/文本匹配
- **可视化元素选择器**：
  - Step 编辑器中每个 step 行有一个「拾取」按钮
  - 点击后进入 picker 模式，iframe 覆盖 inspector 高亮层
  - 点击 iframe 中元素 → 自动填入 CSS selector → 退出 picker 模式
  - Picker 模式下可按 ESC 取消选择
- **Step 管理增强**：
  - 拖拽排序（使用 `react-beautiful-dnd` 或 HTML5 Drag & Drop）
  - Step 复制/粘贴（单个 step 的 duplicate 功能）
  - Step 分组折叠（多个 step 可编组，可折叠/展开）
- **验收**：
  - 每种新增 step 类型在场景执行时正确工作
  - 拾取按钮点击后 iframe 进入可点击选择状态，点击返回 selector
  - Step 列表可拖拽重新排序
  - Step 分组可折叠/展开
  - 向后兼容：旧场景（仅 click/wait/fill）正常执行

### R3: 录制体验优化

- **目标**：完善录制交互，支持暂停/恢复、实时预览和手动插入
- **优化项**：
  - **暂停/恢复**：录制过程中可暂停（停止捕获步骤），恢复后继续捕获
  - **实时步骤预览**：录制过程中，已录制的步骤列表实时展示，支持滚动查看
  - **手动插入步骤**：录制过程中或录制结束后，可在任意位置插入新步骤
  - **步骤编辑**：录制结束后可编辑已录制步骤的 selector/value
- **体验改进**：
  - 录制状态指示器增强（录制时间、步骤数、暂停状态可视化）
  - 暂停/恢复按钮区分明显（与停止按钮视觉分离）
  - 录制完成后直接进入 step 编辑模式
- **验收**：
  - 录制过程中可暂停/恢复，暂停期间的操作不被录制
  - 录制期间实时看到已捕获步骤列表
  - 录制后可在步骤间插入新步骤
  - 可编辑已录制步骤的 selector 和 value
  - 现有录制流程完全保留

### R4: AI 生成改进

- **目标**：提升 AI 生成步骤的灵活性和可控性
- **优化项**：
  - **可配置模型**：支持在 UI 中选择 AI 模型（当前可用 claude-haiku-4-5，预留扩展）
  - **Step 类型覆盖**：AI 生成的 prompt 更新，支持生成新的 step 类型（scroll/keyboard/hover/select/assert）
  - **Step 精炼**：选中单个或多个 step，用自然语言指令让 AI 修改（如"把这个 click 改成 hover"）
  - **错误重试**：AI 生成结果解析失败时，提供重试/修改提示按钮
- **验收**：
  - AI 生成能产出包含新 step 类型的步骤序列
  - 可精炼选中的 step（局部替换而非全量重新生成）
  - 模型选择在 UI 中可切换（即使当前只有一种可用模型）
  - 解析失败时有重试机制

---

## 约束

- Anti-AI-Slop 原则：UI 简洁，不引入多余装饰
- 不破坏现有功能：元素选择、Sketch 对比、URL 导航、已有场景等保持不变
- inspector 脚本扩展需向后兼容（旧消息类型继续工作）
- Step 类型扩展不破坏 `scenarios.json` 数据格式（新增可选字段）
- 测试全部通过
- 底部 Tab 集成后，全屏场景视图作为备选保留
