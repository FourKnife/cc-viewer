## Phase 23: Steps 执行引擎（click/wait/fill）

**目标**: 扩展 inspector postMessage 协议，支持在 iframe 内执行交互步骤序列

**任务**:
- [ ] 找到 inspector 注入脚本（`inspector-inject.js` 或类似），新增 `run-step` 消息处理：
  - `click`: `document.querySelector(selector)?.dispatchEvent(new MouseEvent('click', { bubbles: true, cancelable: true }))`
  - `wait`: `setTimeout` + 回传 `step-done`
  - `fill`: nativeInputValueSetter + `input` event（兼容 React 受控组件）
- [ ] inspector 脚本新增 `step-done` / `step-error` 回传消息
- [ ] `PagePreview.jsx` — `runScenario` 扩展：导航完成后逐步发送 `run-step`，等待 `step-done` 再发下一步
- [ ] `ScenarioPanel.jsx` — 运行时显示步骤进度（步骤 N/M）
- [ ] `ScenarioPanel.jsx` — 表单支持编辑 steps 列表（类型选择 + 参数输入）

**关键文件**:
- inspector 注入脚本（代理注入的 JS 文件）
- `src/components/VisualEditor/PagePreview.jsx`
- `src/components/VisualEditor/ScenarioPanel.jsx`

**预计改动量**: 中
**依赖**: Phase 22

## 里程碑成功标准

1. SideMenu 出现「场景」菜单项，点击后展示 Scenario 面板
2. 可新增 scenario（名称 + URL + storage 键值对），保存到 `.cleffa/scenarios.json`
3. 点击运行，iframe 自动注入 localStorage 并导航到目标 URL
4. steps 中的 click/wait/fill 能正确在 iframe 内执行
5. 批量截图生成画廊，可浏览所有页面状态
6. 固定 scenario 后，iframe 刷新自动重跑
7. 全部测试通过
