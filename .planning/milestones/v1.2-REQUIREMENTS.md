# Cleffa v1.2 — 需求文档

**里程碑**: M1.2 - 可视化编辑体验增强
**状态**: 进行中
**目标**: 优化可视化编辑模式的默认体验、终端交互、设计稿对比自动化

## 功能需求

### P0 - 必须完成

- [ ] **FR-201: ProjectLauncher 默认值优化**
  - 项目路径默认为启动 ccv 时的 CWD（服务端传递给前端）
  - 启动命令默认为 `npm run mock`（替换当前的 `npm run dev`）
  - 用户仍可手动修改

- [ ] **FR-202: 可视化模式右侧终端替换**
  - 可视化编辑模式下，右侧区域使用 TerminalPanel（xterm）替代当前的 ChatView
  - 复用现有 TerminalPanel 组件，功能与主页面终端一致
  - 支持完整的 Claude Code 对话交互（输入、输出、工具审批等）

- [ ] **FR-203: Sketch 设计稿对比 + 自动调整**
  - 选中页面元素后，与 Sketch 设计稿进行对比
  - 对比发现不一致时，自动通过 tmux 向 Claude Code 发送调整命令
  - 命令内容：告知 CC 选中的元素需要与设计稿保持一致，要求调整代码

## 非功能需求

- [ ] **NFR-201: 终端复用兼容性** — TerminalPanel 在可视化模式下的布局需适配 400px 宽度面板
- [ ] **NFR-202: 向后兼容** — 不破坏现有 chat/raw 视图模式和 M1.1 功能
- [ ] **NFR-203: Sketch MCP 降级** — Sketch MCP 未连接时，对比功能优雅降级（提示用户连接）
