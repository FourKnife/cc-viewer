# M1.2: 可视化编辑体验增强 — 路线图

**版本**: v1.2
**状态**: 进行中
**Phases**: 3 (Phase 9-11)

---

## Phase 9: ProjectLauncher 默认值优化

**目标**: 项目路径自动填充 CWD，启动命令默认 `npm run mock`

**任务**:
- [ ] 服务端获取 CWD 并通过 API 或 SSE 初始化事件传递给前端
- [ ] ProjectLauncher 组件接收 CWD 作为 projectPath 的默认值
- [ ] 将 command 默认值从 `npm run dev` 改为 `npm run mock`
- [ ] 添加对应 i18n 条目（如有新文案）

**关键文件**:
- 修改: `ProjectLauncher.jsx`, `AppBase.jsx`, 服务端入口

**预计改动量**: 小

---

## Phase 10: 可视化模式右侧终端替换

**目标**: 可视化编辑模式右侧从 ChatView 替换为 TerminalPanel（xterm 终端）

**任务**:
- [ ] 可视化模式布局中，将右侧 ChatView 替换为 TerminalPanel 组件
- [ ] 确保 TerminalPanel 在 400px 宽度面板中正常渲染和自适应
- [ ] 保留选中元素 Tag 功能 — 选中元素信息需传递给 TerminalPanel（可通过自动注入输入实现）
- [ ] 确保 TerminalPanel 的所有功能正常：输入、输出、工具审批、文件上传等
- [ ] 清理可视化模式下不再需要的 ChatView 相关 props 和逻辑

**关键文件**:
- 修改: `App.jsx`（visual 模式布局区域）, `TerminalPanel.jsx`（宽度适配）
- 可能修改: `App.module.css`（布局样式）

**预计改动量**: 中

**决策要点**:
- TerminalPanel 已有 tmux 会话连接能力，直接复用
- 选中元素信息可通过 tmux send-keys 方式注入到终端

---

## Phase 11: Sketch 设计稿对比 + 自动调整命令

**目标**: 选中元素后与 Sketch 设计稿对比，自动向 CC 发命令调整代码

**任务**:
- [ ] 选中页面元素时，自动获取 Sketch 当前选中图层的截图（通过 Sketch MCP）
- [ ] 获取 iframe 中选中元素区域的截图
- [ ] 对比两张截图，判断是否存在差异（复用已有 ScreenshotCompare / pixelmatch 能力）
- [ ] 当检测到差异时，自动构建调整命令（包含元素信息、设计稿参考、期望行为）
- [ ] 通过 tmux send-keys 将命令发送到右侧 TerminalPanel 的 Claude Code 会话
- [ ] Sketch MCP 未连接时显示提示信息，优雅降级

**关键文件**:
- 修改: `PagePreview.jsx`（截图触发）, `StatusBar.jsx`（对比入口）, `App.jsx`（流程串联）
- 复用: `ScreenshotCompare.jsx`（截图对比逻辑）
- 新建: 可能需要一个 `autoCompare` 工具函数

**预计改动量**: 大

**依赖**:
- Phase 10 完成（右侧需要是 TerminalPanel 才能通过 tmux 发命令）

---

## Phase 12: 结构化设计稿对比 (DOM vs Sketch JSON)

**目标**: 将 Phase 11 的截图像素对比替换为 DOM computed styles vs Sketch 图层 JSON 的结构化属性对比

**任务**:
- [ ] 扩展 inspector-inject.js 的 computedStyle 属性集（新增 fontWeight, lineHeight, padding, margin, borderRadius 等）
- [ ] 新增服务端 `/api/sketch-layer-styles` API，提取 Sketch 图层结构化 JSON（文字样式、填充、边框、阴影）
- [ ] 重写 PagePreview.jsx 的 handleAutoCompare，用结构化属性对比替代 pixelmatch
- [ ] 实现颜色归一化、fontWeight 映射、alignment 映射等转换逻辑
- [ ] 构建 Markdown 差异表格 Prompt 发送到终端

**关键文件**:
- 修改: `public/inspector-inject.js`, `server.js`, `PagePreview.jsx`, `i18n.js`

**预计改动量**: 中

**依赖**: Phase 10 (右侧 TerminalPanel)

---

## 里程碑成功标准

1. 打开可视化编辑模式后，项目路径已自动填充，命令为 `npm run mock`，无需手动输入
2. 右侧是完整的 xterm 终端，可以与 Claude Code 进行正常对话
3. 选中页面元素后，自动与 Sketch 设计稿对比，发现差异时自动向 CC 发送调整命令
